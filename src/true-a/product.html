TrueAutomation solves 2 fundamental problems of existing Selenium-based testing.

Web UI-elements recognition.

Based on customer surveys, Testing Automation teams spend up to 40% of their time on tests maintenance, dealing with weird looking locators, finding elements again and again after each deploy. We’re eliminating this problem.

Test coverage of Technologies with dynamic DOM tree.

It’s challenging to cover technologies like React, JWT, Vaadin with automatic tests. ID’s, class names and many other attributes change either with every deploy or even with every interaction which makes it almost impossible to have decent coverage. TrueAutomation makes it possible now.

Things are done in a way you’re used to

We didn’t want you to switch from familiar environment, to rewrite you previous tests from scratch and spend time on learning new technologies. That’s why TrueAutomation supports all your Selenium-based tests, all methods and functions that you used in regular Selenium right from the box. The list of supported languages and frameworks is constantly expanding as well.

  • compatibility with existing Selenium-based projects
  • support of all functions and commands you would use in Selenium
  • multiple-language and frameworks support
  • minor adjustments required to use your existing projects with TrueAutomation

Constant elements recognition

The problem of locators in existing Selenium solutions has multiple workarounds. Some companies require developers to enter IDs manually before every release, some constantly maintain their tests. In any case, it is not hard, but it is time consuming and not always possible to find a workaround. TrueAutomation is AI-based. So you need only one, initial recognition of the element and we will be finding it for you automatically every time you run your test. This means less time spent on maintenance = faster releases or more time left to cover new functionality.

  • AI-based analysis of multiple attributes of an element
  • Unique element footprint creation
  • Neural network for more precise recognition in case of 2 or more similar elements
  • elements need to be defined only once when you write test for the 1st time
  • further constant automatic recognition of elements
